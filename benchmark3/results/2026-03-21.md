### Key Differences

| Aspect | fastjson3 | wast | Impact |
|--------|-----------|------|--------|
| Object serialization | 19.3% `writeFields` | 25.1% `ObjectImpl.serialize` | wast more focused |
| Collection handling | No hotspot | 9.6% `writeCollection` | wast optimized |
| Memory write | 14.3% `Unsafe.putShort` | - | fastjson3 uses low-level API |
| Long write | Hidden in writeAny | 4.6% `writeLong` | wast has dedicated method |
| String write | 2.8% `writeLatinStringNoCapCheck` | 3.6% `writeLatinJSONString` | Similar |

### Performance Gap Analysis

**Why wast leads by ~15%:**

1. **Collection Optimization**
   - wast has dedicated `writeCollection` method (9.6%)
   - Specialized handling for collection types

2. **Serialization Architecture**
   - wast uses `JSONTypeSerializer` pattern with type dispatch
   - fastjson3 uses generic `writeFields` loop

3. **Method Inlining**
   - Same JIT inline ratio (50%)
   - wast's remaining code may be easier to optimize

4. **Type Specialization**
   - wast has independent `writeLong`, `writeDouble` methods
   - More granular type handling

### Optimization Recommendations

1. **Collection Write** - Add specialized write paths for List/Set
2. **Type Specialization** - Consider dedicated write methods for long/double
3. **Architecture** - Evaluate `JSONTypeSerializer` pattern feasibility
